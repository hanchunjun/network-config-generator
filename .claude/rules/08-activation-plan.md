## 通用全局统一规范（所有方案共用・不可修改）

### 1. 机器码生成标准

- **采集硬件**：CPU 序列号 + 硬盘物理序列号
- **加密方式**：MD5 加密
- **输出格式**：32 位大写唯一机器码
- **硬件绑定特性**：
  - 重装系统、修改系统设置、更新驱动 → 不改变机器码
  - 更换 CPU / 硬盘 → 直接变更机器码，原激活自动失效
- **作用**：实现「一机一软件授权」，杜绝复制、搬运、批量破解

### 2. 软件激活码生成标准

- **运算公式**：用户设备机器码 + 软件内置固定私有密钥 → 统一加密
- **输出格式**：16 位大写激活码
- **核心特性**：
  - 一机一码、不可通用
  - 密钥内置在程序底层，不对外暴露
  - 用户端、管理员制码工具密钥完全一致

### 3. 软件启动强制校验规则

- 软件启动最高优先级执行激活校验
- **未激活状态**：锁定软件全部功能，无法跳过、无法进入主页、无法试用、无绕过入口
- **已激活状态**：正常进入主程序，所有功能开放
- 全程无任何自动试用、临时体验、跳过激活逻辑

### 4. 统一用户激活弹窗（定稿文案 + 界面）

**弹窗标题**：软件未激活・请完成正版激活

**弹窗正文**：
```
当前软件暂未完成授权激活，暂时无法正常使用全部功能。

【本机设备唯一机器码】（程序自动读取并展示 32 位机器码 + 一键复制按钮）

操作指引：
1. 点击一键复制完整机器码
2. 将机器码发送给管理员【老韩】申领软件激活码
3. 填写管理员发放的激活码，点击激活即可正常使用软件

温馨须知：
- 本软件绑定单台设备使用，更换电脑需重新申请激活
- 无自助开通、无自助付费通道，全部由管理员人工统一发放权限
```

**弹窗固定组件**：
- 机器码只读展示框（不可编辑）
- 一键复制机器码按钮
- 激活码输入框
- 立即激活按钮
- **无关闭按钮、无跳过按钮、无试用入口**

### 5. 对外统一用户话术

```
本软件采用设备绑定激活模式，无自动开通功能。首次使用请复制软件内设备机器码发送管理员，审核通过下发专属激活码，完成激活即可正常使用。更换设备使用，需重新提交机器码办理激活。
```

## 方案 A：纯离线永久激活方案

### 核心架构

全程无网络、无服务器、无云端校验、无黑名单、零后台。激活成功后永久有效，断网、重启、重装软件均不失效。管理员无法远程收回权限。

### 用户端完整流程

1. 用户启动软件 → 自动读取硬件生成唯一机器码
2. 强制弹出激活窗口，锁定所有软件功能
3. 用户复制机器码，发送给管理员申领激活码
4. 用户填入管理员下发的激活码，点击激活
5. 软件本地加密存储激活授权信息
6. 激活永久生效，不受时间、网络、重启影响
7. 更换设备 = 新机器码 = 原激活自动失效

### 管理员管控流程

- 建立本地台账：记录「用户、机器码、授权日期、备注」
- 接收机器码 → 使用管理员制码工具生成激活码 → 下发
- 管控逻辑：只控制发不发码，已发码无法收回

### 本地校验规则

- 授权文件加密存储，禁止明文
- 每次启动自动校验：本机机器码 + 激活算法一致性
- 校验失败强制退回激活界面

### 优缺点

| 优点 | 缺点 |
|------|------|
| 零成本、零维护、零报错 | 授权发放后无法收回、无法封禁、无法失效 |
| 100% 离线 | |
| 代码最简单、最稳定 | |

### 适用场景

熟人内部使用、小范围私发、不需要后期管控权限

## 方案 B：离线使用 + 180 天静默联网黑名单可控激活方案（主推）

### 核心架构

99% 时间全程离线使用。软件每 180 天（半年）后台静默联网一次校验云端黑名单。管理员可随时远程收回、封禁、解绑用户软件权限。

### 核心原理

- 云端存放免费静态 TXT 黑名单（Gitee/GitHub Pages，永久免费）
- 文件内仅存放需要封禁的设备机器码
- 管理员添加机器码 = 封禁该设备软件使用权
- 管理员删除机器码 = 恢复使用权
- 软件本地记录校验时间，每 180 天自动静默联网校验一次

### 完整运行流程

#### 日常激活流程（与方案 A 完全一致）

用户出码 → 管理员制码 → 本地离线激活 → 日常离线使用

#### 180 天静默校验机制

1. 软件本地记录「上次联网校验时间」
2. 每满 180 天后台自动静默联网
3. 无界面、无弹窗、不打扰用户
4. **联网失败 / 无网络 / 网络异常**：直接跳过，不判失效
5. 联网成功后比对本地机器码与云端黑名单：
   - **不在黑名单**：刷新 180 天计时，继续离线使用
   - **在黑名单**：立即锁死全部软件功能，弹出失效窗口

#### 管理员权限回收流程

1. 获取违规/需要解绑的用户机器码
2. 录入云端黑名单并保存
3. 用户最晚 180 天内联网一次即自动失效
4. 删除黑名单即可恢复授权

### 授权失效弹窗定稿文案

**弹窗标题**：软件授权已失效

**弹窗正文**：
```
当前设备对应的软件使用权限已被管理员收回，无法继续正常使用。
如需重新开通软件使用权限，请联系管理员【老韩】重新审核办理。
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 超长 180 天周期，用户几乎无感 | 最长 180 天用户必须联网一次，权限才会失效 |
| 平时完全离线，不依赖网络 | |
| 支持远程封禁、解绑、收回权限 | |
| 零服务器费用、零年费、零维护 | |
| 可完美兼容方案 A 激活逻辑 | |

### 适用场景

对外分发、商用试用、学员分发、需要可控、防跑路、防倒卖、可随时解绑

## 方案 C：管理员独立制码工具完整方案

### 工具定位

- 独立 EXE、单独程序、绝不和用户软件合并
- 仅限管理员【老韩】本地使用，禁止外发
- 内置与用户端软件完全一致的私钥与算法

### 工具核心作用

- 接收用户 32 位机器码
- 一键生成 16 位正版软件激活码
- 自动记录授权台账
- 支持方案 B 黑名单快速录入

### 工具固定界面组件

- 机器码粘贴输入框
- 【一键生成激活码】按钮
- 激活码结果展示区
- 【一键复制激活码】按钮
- 用户信息备注栏（姓名 / 用途 / 备注）
- 本地自动保存台账功能
- 黑名单快捷添加按钮（适配方案 B）

### 标准使用流程

1. 用户发来机器码
2. 管理员粘贴至制码工具
3. 一键生成激活码
4. 一键复制发送用户
5. 填写用户信息，自动存档
6. 如需封禁，一键加入黑名单

### 强制权限隔离（绝对重点）

| 用户端软件 | 管理员工具 |
|------|------|
| 只能读码、填码、校验 | 全网唯一可生成激活码的程序 |
| 不能生成任何激活码 | 私钥与用户端完全一致 |

## 开发硬性要求

1. 用户软件、管理员制码工具两个独立 EXE，分开打包
2. 两端私钥、加密算法必须完全一致（共用同一份核心代码）
3. 制码工具禁止任何外流、禁止被用户获取
4. 激活相关文件全部加密存储，禁止明文
5. 所有文案严格按定稿内容实现，不得擅自修改
6. 激活弹窗无关闭按钮、无跳过按钮、无试用入口
7. 方案 B 黑名单校验失败直接跳过，不判失效
