# 使用建议 & 故障排查（V0.3.5）

> 本文件收纳 CLAUDE.md 使用建议章节，面向运维工程师日常使用参考。

## 首次使用

1. 双击 `NetOps.exe` 启动（首次会自动创建所有目录）
2. **登录认证**：首次运行自动生成 `config/account.json`，默认账户 admin/admin，登录成功进入主窗口
3. **试用模式**：未激活可直接使用「锐捷接入交换机配置」和「📜 命令生成」功能，其他功能受限
4. **激活软件**：点击右上角「关于」→「软件激活」→ 复制机器码 → 发送给管理员【老韩】（QQ:223518 / WeChat:tachlaohan）→ 填入激活码 → 点击激活
5. 激活后进入「📁 新建项目」→ 创建项目 → 添加设备清单
6. 进入「⚙ 模型设置」→ 配置AI模型 → 测试连通性 → 保存
7. 进入「🔍 单点运维」→ 添加设备 → 测试连接 → 执行巡检/备份
8. 进入「🔧 项目运维」→ 选择项目 → 执行批量任务 → 查看结果Tab

## AI 使用路径

### 快速分析（第一层：嵌入式）
- **单点/项目运维页** → 选中文件 → 点 `[🔍AI合规巡检]` 或 `[🩺AI故障诊断]` → 自动执行 → 结果归档

### 深度分析（第二层：Agent快捷指令）
- **专家工作站** → 选文件 → 点 `[🔍一键合规巡检]` 或 `[🩺一键故障诊断]` → Prompt自动填入（可编辑修改）→ 点击 `[▶发送分析]` → 查看结果 → `[💾保存报告]` 或 `[📤导出TXT]`

### 自由对话（第三层）
- **专家工作站** → 💬自由对话Tab → 输入任意问题或粘贴内容 → 点击 `[▶发送分析]` → 支持多文件对比分析、多轮迭代提问

## 数据安全

- AI配置自动加密存储为 `.enc` 文件
- 密钥绑定机器ID，拷贝到其他电脑需重新配置
- 所有操作写入日志文件 `logs/netops_YYYYMMDD.log`
- 删除/导出等危险操作有确认对话框
- 密码存储复用 AES-256-GCM 加密体系，密文存储

## 性能预期

- 启动时间：3-5秒
- 配置生成：首次较慢（缓存生效后快速）
- 内存占用：150-200MB
- 文件保存：原子操作，防止损坏
- EXE大小：48.6MB（PyInstaller打包）
- 命令生成性能：万条命令生成 < 500ms

## AI请求优化（V0.2.1）

- **双层分析**：本地规则引擎预检 + AI精审，减少AI无效推理
- **精准上下文提取**：`extract_relevant_context()` 根据异常位置提取周边上下文，替代全文截断
- **Token压缩**：合规审计从 18.4KB → 5.0KB（-73%），故障诊断从 151.6KB → 5.5KB（-96%）
- **Agent文件精简**：reviewer 84%缩减，troubleshooter 84%缩减

## 故障排查

| 故障现象 | 排查步骤 |
|---------|---------|
| 启动崩溃 | 检查 `logs/` 目录下最新日志 + `logs/crash.log` |
| 试用模式限制 | 未激活用户仅可使用「锐捷接入交换机配置」+「📜 命令生成」，属正常设计 |
| 激活弹窗无法关闭 | 试用模式下可点击「稍后再说」关闭；首次启动则必须完成激活 |
| 提示"授权已失效" | 设备被列入黑名单，联系管理员【老韩】重新审核 |
| AI按钮报"未配置模型" | 进入模型设置确认已保存配置并设为激活 |
| 配置丢失 | 检查 `config/ai_config.json.enc` 是否存在 |
| 项目找不到 | 检查 `config/projects_config.json` 和 `projects/` 目录 |
| AI分析崩溃 | 确保 `agents/` 目录下两个 Agent 文件存在 |

## 数据迁移

- 旧版本密码自动迁移到V0.2格式
- 密钥文件 `config/key_info.json` 需妥善保管
- 拷贝到其他电脑需重新配置AI模型（密钥绑定机器ID）

## 注意事项

1. **安全警告**：导出文件包含明文密码，请妥善保管
2. **密钥备份**：`config/key_info.json` 是解密关键，建议备份
3. **日志管理**：`logs/` 目录会积累日志，定期清理
4. **自动化仅限只读**：不自动修改设备配置，高危操作生成人工清单
5. **管理员制码工具**严禁外发给用户端
