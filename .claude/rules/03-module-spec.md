# 六大菜单模块极致细化功能说明（V0.2.0）

## 模块1：📁 新建项目（项目总览）

### 核心功能

1. 一键生成标准化项目目录（含config/output/report/config_backup及子目录）
2. 项目下拉快速切换，绑定当前运维上下文
3. 可视化编辑 device_list.txt 设备清单
4. **V0.2新增：项目总览卡片网格** — 每个项目独立卡片，显示：
   - 🟢🟡🔴 状态灯（根据最近巡检状态）
   - 设备数量、最近备份时间、最近巡检时间、故障设备数
   - [进入项目] 按钮一键切换
5. **V0.2新增：全局统计栏** — 项目数/设备总数/最近全量巡检时间
6. 项目重命名、删除、打开目录

### 交互逻辑

- 未选项目时拦截运维与AI功能并引导新建
- 项目卡片点击「进入项目」自动切换上下文
- 项目目录命名与路径禁止随意改动移动

## 模块2：🔧 项目运维（批量任务中心）

### 核心功能

1. **顶部公共区**：项目下拉选择器（所有任务面板共用）+ 刷新项目按钮
2. **3个任务卡片**：
   - 💾 批量配置备份 — 读取设备清单，SSH批量抓取配置归档
   - 🔍 全网自动化巡检 — 执行CPU/内存/接口/日志只读巡检，生成总报告
   - 🛠 故障设备二次核查 — 自动识别故障IP、执行只读指令、生成人工清单
3. **4个结果Tab**（左右分栏）：
   - 💾 备份文件 — 文件列表 + 内容预览 + `[🔍AI合规巡检]`按钮
   - 📊 巡检报告 — 文件列表 + 内容预览 + `[🩺AI故障诊断]`按钮
   - 🩺 诊断报告 — 文件列表 + 内容预览 + `[🩺AI故障诊断]`按钮
   - 🔍 合规审计 — 文件列表 + 内容预览 + `[🩺AI精审]`按钮
4. **文件管理三件套**：每个Tab列表上方 `[🔄刷新][📂打开][🗑删除]`
5. **AI结果自动归档**：AI诊断→report/diagnosis/，AI审计→report/compliance/
6. **任务联动**：执行批量备份→自动跳转备份Tab并刷新列表

### 交互逻辑

- 必须先选择项目，否则拦截所有操作
- 任务完成后自动刷新对应Tab文件列表
- 所有AI按钮自动匹配当前激活模型
- QThread保存为实例变量，防止GC回收崩溃

## 模块3：🔍 单点运维（单设备全链条）

### 核心功能

1. **工具栏**：`[▶批量巡检] [💾批量备份] [🧪测试连接]`
2. **设备清单**：多设备录入/编辑/删除/IP格式验证/粘贴导入
3. **5个结果Tab**（左右分栏）：
   - 巡检日志 — 实时巡检输出
   - 配置备份 — 备份文件列表 + `[🔍AI合规巡检]`按钮
   - 巡检报告 — 报告列表 + `[🩺AI故障诊断]`按钮
   - 🩺诊断报告 — 诊断报告列表 + `[🩺AI故障诊断]`按钮
   - 🔍合规审计 — 备份文件列表→审计结果 + 本地预检摘要 + `[🩺AI精审]`按钮
4. **文件管理三件套**：每个Tab列表上方 `[🔄刷新][📂打开][🗑删除]`
5. **测试连接**：Ping + SSH/Telnet 双协议连通性验证

### AI能力（第一层：嵌入式自动）

- 🔍 AI合规巡检：选中备份文件→加载reviewer Agent→自动发送→报告归档
- 🩺 AI故障诊断：选中报告→加载troubleshooter Agent→自动发送→报告归档
- 🩺 AI精审：双层审计（本地规则引擎预检 + AI精审）

### 交互逻辑

- 独立运行不干扰批量流程
- 无需录入设备清单，支持临时应急排查
- 数据自动归入EXE同级目录
- 所有AI按钮自动匹配当前激活模型

## 模块4：🤖 专家工作站（深度AI分析）

### 核心功能

1. **左侧面板**：
   - 输入文件区：拖拽/选择/批量选择文件 + 清空
   - 最近文件列表：自动记录最近10个文件，双击添加
   - 历史报告归档：扫描所有项目report目录 + `[🔄][📂][🗑]`三件套
2. **右侧3个Tab**：
   - 💬 自由对话：完全自定义Prompt + 多轮迭代
   - 📋 合规审计：加载reviewer Agent（可编辑）+ 多文件输入
   - 🩺 故障诊断：加载troubleshooter Agent（可编辑）+ 多文件输入
3. **Agent快捷指令按钮**：
   - `[🔍 一键合规巡检]`：加载Agent→预填Prompt→可编辑→手动发送
   - `[🩺 一键故障诊断]`：加载Agent→预填Prompt→可编辑→手动发送
4. **结果输出**：Markdown渲染 + `[💾保存报告][📤导出TXT]`

### 与嵌入式AI的区别

| 维度 | 嵌入式AI（单点/项目运维） | 专家工作站 |
|------|------|------|
| 触发方式 | 点按钮立即执行 | 预填Prompt→可编辑→手动发送 |
| 文件数量 | 单文件自动绑定 | 支持多文件手动选择 |
| Prompt | 不可编辑（自动拼接） | 完全可编辑 |
| 多轮对话 | 不支持 | 支持 |

### 交互逻辑

- 依赖系统AI配置，未配置则禁用发送
- 多文件内容自动拼接分隔符
- 保存/导出路径可自由选择

## 模块5：🖥 设备配置（配置生成器）

### 核心功能

1. 四大厂商切换（锐捷/华为/H3C/思科）
2. 四类设备类型切换（接入交换机/核心交换机/路由器/AC控制器）
3. 动态加载对应配置表单页面
4. 参数可视化填写，自动生成合规配置脚本
5. 脚本预览、本地保存
6. **新增：** 配置缓存机制（LRU缓存，128条目）
7. **新增：** 输入参数统一验证和命令注入防护

### 交互逻辑

- 完全沿用原有PyQt5界面与调度逻辑
- 与项目模块解耦，可独立使用

## 模块6：⚙ 模型设置（AI配置管理）

### 核心功能

1. AI模型多配置管理：新增/删除/重命名/切换
2. 加密存储：SecureConfigFile + AES-GCM
3. 连通性测试：验证API可用性
4. **自动匹配**：三个业务页面（单点运维/项目运维/专家工作站）的所有AI按钮自动读取当前激活的模型配置，切换即生效

### AI配置自动匹配机制

```
模型设置页 → 用户选哪个模型，写入 active_profile
                ↓
get_active_ai_config()  ← 单点运维页（3个AI按钮）
                        ← 项目运维页（3个AI按钮）
                        ← 专家工作站（所有AI按钮）
```

## AI能力三层架构总结

```
第一层：嵌入式AI按钮（单点运维 + 项目运维）
       自动绑定上下文、一键执行、不可编辑Prompt
       
第二层：Agent快捷指令（专家工作站专属）
       手动选文件 → 加载Agent → 可编辑Prompt → 手动发送
       
第三层：自由对话（专家工作站专属）
       完全自定义Prompt + 多文件输入 + 多轮迭代
```
