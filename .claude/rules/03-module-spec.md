# 七大菜单模块极致细化功能说明（V0.4.1）

## 模块1：📁 新建项目（项目总览）

### 核心功能

1. 一键生成标准化项目目录（含config/output/report/config_backup及子目录）
2. 项目下拉快速切换，绑定当前运维上下文
3. 可视化编辑 device_list.txt 设备清单
4. **V0.2新增：项目总览卡片网格** — 每个项目独立卡片，显示：
   - 🟢🟡🔴 状态灯（根据最近巡检状态）
   - 设备数量、最近备份时间、最近巡检时间、故障设备数
   - [进入项目] 按钮一键切换
5. **V0.2新增：全局统计栏** — 项目数/设备总数/最近全量巡检时间
6. 项目重命名、删除、打开目录

### 交互逻辑

- 未选项目时拦截运维与AI功能并引导新建
- 项目卡片点击「进入项目」自动切换上下文
- 项目目录命名与路径禁止随意改动移动

## 模块2：🔧 项目运维（批量任务中心）

### 核心功能

1. **顶部公共区**：项目下拉选择器（所有任务面板共用）+ 刷新项目按钮
2. **3个任务卡片**：
   - 💾 批量配置备份 — 读取设备清单，SSH批量抓取配置归档
   - 🔍 全网自动化巡检 — 执行CPU/内存/接口/日志只读巡检，生成总报告
   - 🛠 故障设备二次核查 — 自动识别故障IP、执行只读指令、生成人工清单
3. **4个结果Tab**（左右分栏）：
   - 💾 备份文件 — 文件列表 + 内容预览 + `[🔍AI合规巡检]`按钮
   - 📊 巡检报告 — 文件列表 + 内容预览 + `[🩺AI故障诊断]`按钮
   - 🩺 诊断报告 — 文件列表 + 内容预览 + `[🩺AI故障诊断]`按钮
   - 🔍 合规审计 — 文件列表 + 内容预览 + `[🩺AI精审]`按钮
4. **文件管理三件套**：每个Tab列表上方 `[🔄刷新][📂打开][🗑删除]`
5. **AI结果自动归档**：AI诊断→report/diagnosis/，AI审计→report/compliance/
6. **任务联动**：执行批量备份→自动跳转备份Tab并刷新列表

### 交互逻辑

- 必须先选择项目，否则拦截所有操作
- 任务完成后自动刷新对应Tab文件列表
- 所有AI按钮自动匹配当前激活模型
- QThread保存为实例变量，防止GC回收崩溃

3. **5个结果Tab**（左右分栏）：
   - 巡检日志 — 实时巡检输出
   - 配置备份 — 备份文件列表 + `[🔍AI合规巡检]`按钮
   - 巡检报告 — 报告列表 + `[🩺AI故障诊断]`按钮
   - 🩺诊断报告 — 诊断报告列表 + `[🩺AI故障诊断]`按钮
   - 🔍合规审计 — 备份文件列表→审计结果 + 本地预检摘要 + `[🩺AI精审]`按钮
4. **文件管理三件套**：每个Tab列表上方 `[🔄刷新][📂打开][🗑删除]`

### AI能力（第一层：嵌入式自动）

### 交互逻辑

- 独立运行不干扰批量流程
- 无需录入设备清单，支持临时应急排查

- 所有AI按钮自动匹配当前激活模型

## 模块4：🤖 专家工作站（深度AI分析）

### 核心功能

1. **左侧面板**：
   - 输入文件区：拖拽/选择/批量选择文件 + 清空
   - 最近文件列表：自动记录最近10个文件，双击添加
   - 历史报告归档：扫描所有项目report目录 + `[🔄][📂][🗑]`三件套
2. **右侧3个Tab**：
   - 💬 自由对话：完全自定义Prompt + 多轮迭代
   - 📋 合规审计：加载reviewer Agent（可编辑）+ 多文件输入
   - 🩺 故障诊断：加载troubleshooter Agent（可编辑）+ 多文件输入
3. **Agent快捷指令按钮**：
   - `[🔍 一键合规巡检]`：加载Agent→预填Prompt→可编辑→手动发送
   - `[🩺 一键故障诊断]`：加载Agent→预填Prompt→可编辑→手动发送
4. **结果输出**：Markdown渲染 + `[💾保存报告][📤导出TXT]`

### 与嵌入式AI的区别

| 维度 | 嵌入式AI（单点/项目运维） | 专家工作站 |
|------|------|------|
| 触发方式 | 点按钮立即执行 | 预填Prompt→可编辑→手动发送 |
| 文件数量 | 单文件自动绑定 | 支持多文件手动选择 |
| Prompt | 不可编辑（自动拼接） | 完全可编辑 |
| 多轮对话 | 不支持 | 支持 |

### 交互逻辑

- 依赖系统AI配置，未配置则禁用发送
- 多文件内容自动拼接分隔符
- 保存/导出路径可自由选择

## 模块7：📜 命令生成（批量脚本工具）★V0.3.1新增 → V0.3.2重构

### 核心功能

1. **命令模板编辑**：
   - 多行文本编辑区编写命令模板
   - 支持5组参数占位符：`%a`, `%b`, `%c`, `%d`, `%e`
   - 实时预览生成的命令结果
2. **参数配置面板（ParamGroupWidget × 5）**：
   - 每个参数独立配置：基数(base)、步长(step)
   - 循环勾选框 `loop_cb`：启用后该参数参与差异化生成
3. **互斥双模式生成引擎（V0.3.2重构）**：
   - **固定重复模式**：勾选"重复次数"→ `loop_count` 统一驱动，所有参数值展开到 `loop_count`
   - **差异化生成模式**：勾选参数循环→各参数按各自范围独立展开，同步索引生成
   - 双向互斥联动：`loop_cb` ↔ `cmd_count_cb`（`blockSignals` 防信号循环）
   - `mode_hint_label` 实时显示当前模式
4. **模板管理4按钮（V0.3.2改造）**：
   - 📝新增模板(蓝色) / ✏️重命名(橙色) / 💾保存模板(绿色) / 🗑删除模板(灰色黑字)
   - 平铺独立按钮，非下拉菜单
   - `_update_template_buttons()` 按模板类型（预置/用户）启用/禁用
5. **预置模板库**：
   - 内置常用网络设备命令模板（VLAN创建、接口配置、ACL规则等）
   - 一键加载预置模板，快速开始
6. **结果操作**：
   - 生成结果实时显示在文本框
   - 一键复制全部生成内容
   - 导出为.txt文件保存

### 技术实现

- **文件位置**：`src/ui/batch_cmd_generator_page.py`
- **核心类**：`BatchCmdGeneratorPage(QWidget)` + `ParamGroupWidget(QGroupBox)`
- **V0.3.2关键信号**：`ParamGroupWidget.loop_toggled` pyqtSignal
- **依赖库**：纯PyQt5 + Python标准库

### 参数系统详解

```
模板示例：
interface GigabitEthernet0/0/%a
 description To_Switch_%b
 ip address 192.168.%c.%d 255.255.255.0

【固定重复模式】（勾选"重复次数"=5）：
→ 5条命令，使用当前编辑区参数值，生成相同命令×5

【差异化生成模式】（勾选%a循环=5, %b循环=5）：
%a: base=1, step=1 → 生成 1,2,3,4,5
%b: base=10, step=1 → 生成 10,11,12,13,14
→ 5条命令：%a/%b同步递增
```

### 核心生成逻辑（V0.3.2重构）

```python
_generate():
  1. 判定模式：cmd_count_cb.isChecked() → 固定重复 / loop_cb勾选 → 差异化
  2. loop_count = 从模式判定取得
  3. for i in range(loop_count):
       固定重复 → values = [base + i * step for i in range(loop_count)]
       差异化生成 → 各参数独立展开，按索引取值
  4. 替换模板中所有 %a~%e 占位符
```

### 交互逻辑

- 模板编辑区和参数配置区左右分栏布局
- 修改参数后点击[生成预览]手动触发
- 占位符语法错误时给出明确提示
- 模板管理按钮 QSS 排除选择器 `QPushButton:not(#id1):not(#id2)...` 避免样式污染
- **试用模式开放**：未激活用户也可正常使用此功能
