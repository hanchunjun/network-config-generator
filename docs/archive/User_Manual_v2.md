## 🎯 产品概述

### 什么是NetOps运维平台？

NetOps是一款免安装、双击直跑的政企网络自动化运维桌面工具。支持锐捷、华为、H3C、思科四大厂商全系列网络设备，覆盖开局建站配置生成、日常批量运维、单点故障排查、AI智能诊断与合规审计全场景。

### 核心特色

- **免安装绿色便携**：单EXE约48MB，整体拷贝即用，U盘随身
- **AI能力三层架构**：一键自动 → Agent快捷指令 → 自由对话，从简单到深度
- **多项目物理隔离**：每个项目数据独立存储，杜绝跨项目混淆
- **全中文报告**：AI输出均为中文，含分级整改建议
- **安全第一**：自动化仅执行只读指令，绝不修改设备配置

### 六大功能模块

```
📁 新建项目   →   🔧 项目运维   →   🔍 单点运维   →   🤖 专家工作站   →   🖥 设备配置   →   ⚙ 模型设置
   建               管                修                研                配              设
```

## 💻 安装与启动

### 系统要求

- **操作系统**：Windows 10/11（64位）
- **内存**：2GB RAM 以上
- **存储**：100MB 可用空间
- **Python**：不需要安装（已内嵌打包）

### 启动方式

1. 双击 `NetworkConfigGenerator.exe`
2. 首次启动会自动创建所有目录，约3-5秒进入主界面
3. 所有数据跟随EXE存储，拷贝整个文件夹即可迁移

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+1 | 📁 新建项目 |
| Ctrl+2 | 🔧 项目运维 |
| Ctrl+3 | 🔍 单点运维 |
| Ctrl+4 | 🤖 专家工作站 |
| Ctrl+5 | 🖥 设备配置 |
| Ctrl+6 | ⚙ 模型设置 |

## 🚀 快速入门

### 第一步：创建项目

1. 点击左侧导航 **📁 新建项目**
2. 点击「＋ 新建项目」→ 输入名称 → 确定
3. 点击「编辑设备清单」→ 按格式录入设备

设备格式：`IP，厂商，设备类型，用户名，密码，协议，特权密码`

### 第二步：配置AI模型

1. 点击 **⚙ 模型设置**
2. 填写Base URL、API Key、模型名
3. 点击「测试连接」验证
4. 点击「保存配置」
5. 支持添加多套配置，激活哪套用哪套

### 第三步：开始使用

- **单台排查** → 🔍 单点运维：录入IP → 测试连接 → 巡检/备份/AI分析
- **批量运维** → 🔧 项目运维：选项目 → 点执行 → 切Tab看结果
- **深度分析** → 🤖 专家工作站：选文件 → Agent快捷指令 → 编辑Prompt → 发送

## 🎛️ 功能模块详解

### 📁 新建项目（项目总览）

**用途**：管理所有项目，全局统览。

**项目卡片总览**：

每个项目一张卡片，显示：
- 🟢🟡🔴 状态灯（绿色正常 / 黄色未巡检 / 红色有故障）
- 设备数量、最近备份时间、最近巡检时间、故障设备数
- 「进入项目」按钮一键切换

**操作按钮**：

| 按钮 | 功能 |
|------|------|
| ＋ 新建项目 | 创建项目，自动生成标准化目录 |
| ✏ 重命名 | 修改项目名称 |
| 🗑 删除 | 删除整个项目（有确认对话框） |
| 📂 打开目录 | 用文件管理器打开项目目录 |
| 编辑设备清单 | 添加/修改/删除设备信息 |

**全局统计栏**：项目总数 / 设备总数 / 最近全量巡检时间。

---

### 🔧 项目运维（批量任务中心）

**用途**：对某个项目执行批量备份、全网巡检、故障核查，管理所有输出文件。

**操作流程**：

1. 顶部下拉选择目标项目
2. 点击任务卡片「▶ 执行」：
   - 💾 **批量配置备份** — SSH批量抓取所有设备配置
   - 🔍 **全网自动化巡检** — 检查CPU/内存/接口/日志
   - 🛠 **故障设备二次核查** — 只读指令深挖故障根因
3. 任务完成后自动跳转对应Tab查看结果

**4个结果Tab**（左右分栏，左侧文件列表 + 右侧内容预览 + AI按钮）：

| Tab | AI按钮 |
|-----|--------|
| 💾 备份文件 | `[🔍AI合规巡检]` |
| 📊 巡检报告 | `[🩺AI故障诊断]` |
| 🩺 诊断报告 | `[🩺AI故障诊断]` |
| 🔍 合规审计 | `[🩺AI精审]` |

---

### 🔍 单点运维（单设备全链条）

**用途**：针对某台设备执行全链条运维操作（巡检/备份/诊断/审计）。

**工具栏**：`[▶批量巡检] [💾批量备份] [🧪测试连接]`

**测试连接**：Ping + SSH/Telnet 双协议连通性验证。

**5个结果Tab**：

| Tab | 内容 | AI按钮 |
|-----|------|--------|
| 巡检日志 | 实时巡检输出 | — |
| 配置备份 | 备份文件列表 + 内容预览 | `[🔍AI合规巡检]` |
| 巡检报告 | 报告列表 + 内容预览 | `[🩺AI故障诊断]` |
| 🩺 诊断报告 | AI诊断报告列表 + 预览 | `[🩺AI故障诊断]` |
| 🔍 合规审计 | 备份文件→审计结果 + 本地预检 | `[🩺AI精审]` |

**使用场景**：临时接入一台设备排查、项目外设备应急维护、新设备上线前检查。

---

### 🤖 专家工作站（深度AI分析）

**用途**：进行深度、多文件、自定义的AI分析。AI能力的最高级入口。

**架构**：左侧文件管理面板 + 右侧3个Tab。

**三个Tab**：

| Tab | 用途 | 特点 |
|-----|------|------|
| 💬 自由对话 | 完全自定义Prompt + 多轮迭代 | 最灵活 |
| 📋 合规审计 | 加载合规审计Agent + 可编辑Prompt | Agent预填 |
| 🩺 故障诊断 | 加载故障诊断Agent + 可编辑Prompt | Agent预填 |

**Agent快捷指令按钮**：

- `[🔍 一键合规巡检]`：加载reviewer Agent → 预填Prompt（可改）→ 手动发送
- `[🩺 一键故障诊断]`：加载troubleshooter Agent → 预填Prompt（可改）→ 手动发送

**与嵌入式AI的区别**：

| 维度 | 嵌入式AI（单点/项目运维） | 专家工作站 |
|------|------|------|
| 触发 | 点按钮立即执行 | 预填→编辑→手动发送 |
| 文件 | 单文件自动绑定 | 支持多文件 |
| Prompt | 不可编辑 | 完全可编辑 |
| 对话 | 单次 | 多轮迭代 |

---

### 🖥 设备配置（配置生成器）

**用途**：为四厂商四类设备生成开局配置脚本。

**操作**：选择厂商 → 选择设备类型 → 填写参数 → 生成脚本 → 预览/保存。

**支持**：锐捷RGOS / 华为VRP / H3C Comware / 思科IOS × 接入交换机 / 核心交换机 / 路由器 / AC控制器。

---

### ⚙ 模型设置（AI配置管理）

**用途**：管理AI大模型配置，支持多套模型切换。

**核心机制**：所有页面的所有AI按钮自动读取当前激活的模型配置，切换即生效，无需重启。

```
模型设置页 → 用户选哪个模型，写入 active_profile
              ↓
get_active_ai_config()  ← 单点运维（3个AI按钮）
                        ← 项目运维（3个AI按钮）
                        ← 专家工作站（所有AI按钮）
```

**操作**：新增配置 / 设为激活 / 重命名 / 删除 / 测试连接。

## 🔄 操作流程指南

### 场景1：新网络开局

1. **📁 新建项目** → 创建"XX局开局项目"
2. 添加设备清单（支持粘贴导入）
3. **🖥 设备配置** → 为每类设备生成开局脚本
4. **🔧 项目运维** → 批量配置备份，建立基线
5. **🔧 项目运维** → 全网巡检，确认状态
6. 切换到合规审计Tab → 点 `[🩺AI精审]` 审计合规性

### 场景2：日常运维巡检

1. **📁 新建项目** → 项目总览卡片确认状态
2. **🔧 项目运维** → 选项目 → 点「▶ 执行」全网巡检
3. 切📊巡检报告Tab → 查看报告
4. 对异常设备 → 切🩺诊断报告Tab → 点 `[🩺AI故障诊断]`
5. 高风险操作生成人工清单处理

### 场景3：单台设备故障排查

1. **🔍 单点运维** → 录入设备IP/账号
2. 点 `🧪测试连接` 验证连通性
3. 点 `▶批量巡检` 执行巡检
4. 切🩺诊断报告Tab → 点 `[🩺AI故障诊断]` AI深度分析
5. 按AI建议进行处理

### 场景4：深度AI分析

1. **🤖 专家工作站** → 选择文件（支持多文件对比）
2. 点 `[🔍一键合规巡检]` 或 `[🩺一键故障诊断]`
3. 编辑Prompt（可自由调整分析方向）
4. 点 `▶发送分析` → 等待结果
5. `💾保存报告` 或 `📤导出TXT`

## 🚨 故障排除

### 问题1：程序无法启动

**解决**：
1. 安装 Visual C++ Redistributable
2. 暂时关闭防病毒软件
3. 以管理员权限运行
4. 检查 `logs/` 下最新日志

### 问题2：设备连接失败

**解决**：
1. 先用 🔍单点运维 的「🧪测试连接」验证Ping和SSH
2. 确认账号密码正确
3. 确认设备SSH/Telnet服务已启用
4. 检查防火墙/ACL限制

### 问题3：AI按钮报"未配置模型"

**解决**：进入 ⚙模型设置 → 确认已保存至少一套配置且设为激活状态。

### 问题4：AI分析崩溃或无响应

**解决**：
1. 确认 `agents/` 目录下两个Agent文件存在
2. 检查模型设置中API连通性
3. 查看 `logs/` 目录下的错误日志

### 问题5：项目找不到

**解决**：检查EXE同级目录下 `projects_config.json` 和 `projects/` 目录。

### 日志文件

- `logs/netops_YYYYMMDD.log` — 按日期分割的运行日志
- 包含操作记录、错误信息、调试输出

## 📊 技术规格

### 性能指标

- 启动时间：3-5秒
- 配置生成：< 2秒/设备（缓存命中后）
- 批量备份：100台设备约5-10分钟
- 巡检速度：100台设备约3-5分钟
- AI分析：单设备约30-120秒

### 容量规格

- 单项目设备：建议< 1000台
- 项目数量：无限制
- 备份保留：手动管理
- EXE大小：约48.7MB

### 安全机制

- **加密算法**：AES-GCM
- **密钥管理**：V2版本，绑定机器ID
- **数据隔离**：多项目物理目录隔离
- **只读指令**：自动化仅执行show/display/ping
- **高危操作**：接口启停/路由变更/配置保存仅生成人工清单
- **原子操作**：文件写入防损坏
- **操作日志**：完整审计记录
