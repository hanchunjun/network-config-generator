============================================================
  1. 产品概述
============================================================
  NetOps 企业网络自动化运维平台是一款面向网络工程师的多厂商网络设备配置脚本生成与自动化运维工具。平台支持锐捷、华为、华三（H3C）、思科（Cisco）四大主流厂商的网络设备，覆盖接入交换机、核心交换机、路由器、AC控制器等常见设备类型。

  --------------------------------------------------
  1.1 核心功能
  --------------------------------------------------
    - 多厂商设备配置脚本一键生成
    - 项目化管理，支持设备清单导入/导出
    - 批量配置备份与全网自动化巡检
    - 单点设备快速巡检与配置备份
    - 基于大模型的AI智能故障诊断与合规审计
    - 故障设备自动识别与二次核查

    - 步骤2：点击目标设备类型（接入交换机/核心交换机/路由器/AC），选中后按钮变为蓝色
    - 步骤3：系统自动加载对应的配置页面，填写设备参数（如主机名、管理IP、VLAN等）
    - 步骤4：点击"生成配置"按钮，系统自动生成标准化配置脚本
    - 步骤5：可复制生成的配置脚本，或保存到项目目录中
  💡 提示：必须先选择厂商，再选择设备类型，两者都选中后才会显示配置页面。

  --------------------------------------------------
  3.2 项目管理
  --------------------------------------------------
  项目管理模块是整个平台的核心组织单元。每个项目对应一个独立的网络工程项目，包含设备清单、配置脚本、巡检报告、配置备份等完整数据。

  >> 3.2.1 创建项目
  操作步骤：
    - 在"新建项目"区域输入项目名称（如：栖霞区电子政务外网项目）
    - 点击"创建项目"按钮
    - 系统自动在 C:\Network-Config\projects\ 目录下创建项目文件夹
    - 项目文件夹包含以下子目录：config（配置）、output（输出）、report（报告）、config_backup（配置备份）

  >> 3.2.2 切换与管理项目
  在"项目切换"区域可以进行以下操作：
    - 切换项目：在下拉框中选择目标项目，点击"切换项目"按钮
    - 刷新列表：点击"刷新列表"按钮更新项目下拉框
    - 编辑项目：选中项目后点击"编辑项目"可修改项目名称
    - 删除项目：选中项目后点击"删除项目"可删除整个项目（含所有数据，操作不可逆）
  ⚠ 注意：删除项目将永久删除项目文件夹及其所有数据，请谨慎操作！

  >> 3.2.3 设备清单管理
  设备清单是运维自动化的数据基础，包含项目中所有网络设备的连接信息。

  --------------------------------------------------
  3.3 运维工具箱
  --------------------------------------------------
  运维工具箱提供三个独立的自动化运维任务，每个任务可独立选择项目，互不干扰。所有操作仅执行只读指令，不会修改设备任何配置。

  >> 3.3.1 批量配置备份
  功能说明：读取项目设备清单，批量SSH登录所有设备，抓取运行配置（running-config），按"设备IP+设备类型"命名存入项目的 config_backup 目录。
  操作步骤：
    - 在"批量配置备份"面板中选择目标项目
    - 点击"执行批量备份"按钮
    - 观察进度条和日志输出，等待任务完成
    - 备份文件保存在 项目目录\config_backup\ 下

  >> 3.3.2 全网自动化巡检
  功能说明：对项目中的所有设备执行CPU、内存、接口状态、日志等只读巡检指令，生成全网巡检总报告，自动识别故障设备并单独归档。
  操作步骤：
    - 在"全网自动化巡检"面板中选择目标项目
    - 点击"执行全网巡检"按钮
    - 系统自动逐台登录设备执行巡检命令
    - 巡检完成后，报告保存在 项目目录\output\ 下
    - 异常设备信息单独归档到 output\single_exception\ 目录

  >> 3.3.3 故障设备二次核查
  功能说明：自动读取上一轮巡检中识别的故障设备IP，执行合规只读指令进行二次核查。高危配置指令自动剥离，生成人工操作清单，确保安全。
  操作步骤：
    - 在"故障设备二次核查"面板中选择目标项目
    - 点击"执行故障核查"按钮
    - 系统仅对故障设备执行核查，节省时间
    - 核查结果保存在 项目目录\output\trouble_check_result\ 下
  ⚠ 注意：所有运维任务仅执行只读命令（show、display等），不会修改设备配置。但请确保使用只读权限的账号。

  --------------------------------------------------
  3.4 单点巡检
  --------------------------------------------------
  单点巡检模块不依赖项目设备清单，可直接输入单台设备的IP和凭证，执行只读巡检。支持一键备份配置并联动AI分析。适用于临时排查、单台设备验收等场景。
  操作步骤：
    - 步骤1：填写设备连接信息（IP地址、协议、厂商、设备类型）
    - 步骤2：填写登录凭证（用户名、密码，Cisco设备需填写特权密码）
    - 步骤3：选择执行操作：
    - 单设备巡检：执行只读巡检命令，输出设备健康状态报告
    - 单设备备份：SSH登录抓取运行配置并保存
    - AI分析结果：将巡检结果发送给AI模型进行智能分析
    - 步骤4：观察进度条和状态提示，等待任务完成
  💡 提示：单点巡检适合快速排查单台设备问题，无需创建项目即可使用。巡检结果可联动AI分析模块进行深度诊断。

  --------------------------------------------------
  3.5 AI 智能分析
  --------------------------------------------------
  AI智能分析模块基于大语言模型，对设备巡检日志进行智能故障诊断，或对设备配置进行合规性审计。使用前请先在"模型设置"中配置AI模型连接。

  >> 3.5.1 界面布局
  AI分析页面包含以下交互区域：
    - 当前项目：下拉框选择项目，支持右键菜单刷新/清除，右侧✕按钮可清除选择
    - 分析模式：下拉框选择"故障诊断"或"配置合规审计"
    - 目标文件：显示当前选中的文件路径，支持浏览选择、清除、最近文件记录
    - 批量模式：支持同时选择多个文件进行分析
    - 输入数据区：支持粘贴文本、拖拽文件（.txt/.log/.md）、从剪贴板粘贴、加载文件
    - 文件预览：自动显示加载文件的前5行内容，方便确认文件正确性
    - 分析结果区：显示AI返回的结构化分析报告，支持保存和导出

  >> 3.5.2 两种分析模式


  >> 3.5.3 操作步骤
    - 步骤1：在"模型设置"中配置好AI模型（首次使用必须）
    - 步骤2：选择分析模式（故障诊断 / 配置合规审计）
    - 步骤3：加载分析数据（三种方式任选）：

============================================================
  4. 典型工作流程
============================================================

  --------------------------------------------------
  4.1 新项目上线流程
  --------------------------------------------------
  以下是一个典型的新网络工程项目使用本平台的完整流程：
    - 第1步：创建项目 → 在"新建项目"模块输入项目名称并创建
    - 第2步：导入设备 → 下载模板，填写设备清单，导入到项目中
    - 第3步：生成配置 → 在"设备配置"模块选择厂商和类型，生成各设备配置脚本
    - 第4步：部署设备 → 将生成的配置脚本部署到实际设备上
    - 第5步：配置备份 → 在"运维工具箱"执行批量配置备份，存档运行配置
    - 第6步：全网巡检 → 执行全网自动化巡检，验证设备运行状态

  --------------------------------------------------
  4.2 故障排查流程
  --------------------------------------------------
    - 第1步：单点巡检 → 在"单点巡检"模块输入故障设备IP，执行快速巡检
    - 第2步：AI诊断 → 将巡检结果粘贴到"AI分析"模块，选择"故障诊断"模式
    - 第3步：查看报告 → AI输出结构化的故障诊断报告，包含原因分析和修复建议
    - 第4步：全网核查 → 如怀疑多台设备受影响，在"运维工具箱"执行全网巡检
    - 第5步：二次确认 → 对故障设备执行"故障设备二次核查"

  --------------------------------------------------
  4.3 合规审计流程
  --------------------------------------------------
    - 第1步：获取配置 → 在"运维工具箱"执行批量配置备份，获取所有设备运行配置
    - 第2步：AI审计 → 在"AI分析"模块选择"配置合规审计"模式，加载配置文件
    - 第3步：整改 → 根据AI审计报告中的建议，逐项整改配置风险
    - 第4步：复检 → 整改完成后再次执行审计，确认所有风险项已处理


============================================================
  5. 常见问题
============================================================
  Q1：程序启动后界面显示异常怎么办？
  请确认您的操作系统为 Windows 10 或 Windows 11，屏幕分辨率不低于 1366×768。如仍有问题，请尝试以管理员身份运行。
  Q2：SSH连接设备失败怎么办？
  请检查：① 设备IP地址是否正确；② 设备是否开启了SSH服务；③ 用户名和密码是否正确；④ 网络是否可达（可使用"连接测试"功能验证）。
  Q3：AI分析功能无法使用？
  请确认已在"模型设置"中正确配置了AI模型参数，并通过了"测试连接"。需要有效的API Key和网络连接。
  Q4：如何批量导入设备清单？
  点击"下载模板"获取标准CSV模板，按模板格式填写设备信息后，点击"导入"按钮选择文件即可。
  Q5：项目删除后能恢复吗？
  项目删除是永久性的，会删除项目文件夹及其所有数据。建议删除前先确认项目数据已备份。
  Q6：运维任务会修改设备配置吗？
  不会。所有运维任务仅执行只读命令（show、display等），不会对设备配置做任何修改。请使用只读权限的账号以确保安全。
  Q7：支持哪些设备型号？
  理论上支持所有支持SSH/Telnet的标准网络设备。已测试的厂商包括锐捷、华为、H3C、思科的主流交换机和路由器。
  Q8：如何获取API Key？
  请访问对应AI厂商的官方网站注册并获取API Key。例如：OpenAI用户访问 platform.openai.com，DeepSeek用户访问 platform.deepseek.com。
