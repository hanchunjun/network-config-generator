**版本：V0.2.1 | 日期：2026年5月20日**

## 一、产品简介

本工具是一款面向政企网络的自动化运维桌面软件，免安装、双击直跑、整体拷贝即用。支持锐捷、华为、H3C、思科全系列网络设备，覆盖开局建站配置生成、日常批量运维、单点故障排查、AI智能诊断与合规审计全场景。

### 核心特点

- **单EXE绿色免安装**：约47MB，拷贝到任何Windows电脑直接双击运行
- **多项目物理隔离**：每个项目数据独立存储，杜绝跨项目混淆
- **AI三层架构**：一键自动分析 → Agent快捷指令 → 自由对话，从简单到深度
- **双层AI分析**：本地规则引擎预检 + AI精审，异常精准定位，Token消耗降低70%+
- **全中文报告**：所有AI输出均为中文，含分级整改建议
- **安全第一**：自动化仅执行只读指令，绝不修改设备配置

## 二、快速上手（5分钟）

### 第一步：启动

双击 `NetworkConfigGenerator.exe`，首次启动会自动创建目录结构，约3-5秒进入主界面。

### 第二步：创建项目

1. 点击左侧导航 **📁 新建项目**
2. 点击 `＋ 新建项目` → 输入项目名称 → 确定
3. 点击 `编辑设备清单` → 按格式录入设备信息
4. 设备格式：`IP，厂商，设备类型，用户名，密码，协议，特权密码`
5. 支持从Excel粘贴导入

### 第三步：配置AI模型

1. 点击 **⚙ 模型设置**
2. 选择厂商 → 填写Base URL → 填写API Key → 填写模型名
3. 点击 `测试连接` 验证连通性
4. 点击 `保存配置`
5. 支持添加多套模型配置，激活哪套用哪套

### 第四步：开始使用

现在可以：
- 进 **🔍 单点运维** 排查某台设备
- 进 **🔧 项目运维** 批量备份/巡检整个项目
- 进 **🤖 专家工作站** 进行深度AI分析

## 三、六大功能模块

### 📁 新建项目（项目总览）

**用途：** 管理所有项目，一目了然查看全局状态。

**操作：**
- `＋ 新建项目`：创建新项目，自动生成标准化目录
- `✏ 重命名`：修改项目名称
- `🗑 删除`：删除整个项目（有确认对话框）
- `📂 打开目录`：用文件管理器打开项目目录
- `编辑设备清单`：添加/修改/删除设备信息

**项目总览卡片：** 每个项目显示状态灯、设备数量、最近备份/巡检时间、故障设备数，点击「进入项目」一键切换。

---

### 🔧 项目运维（批量任务中心）

**用途：** 对某个项目执行批量备份、全网巡检、故障核查，并管理所有输出文件。

**操作流程：**
1. 顶部下拉框选择目标项目
2. 点击任务卡片 `▶执行`：
   - 💾 批量配置备份 — SSH批量抓取所有设备配置
   - 🔍 全网自动化巡检 — 检查CPU/内存/接口/日志
   - 🛠 故障设备二次核查 — 只读指令深挖故障根因
3. 任务完成后，切换到下方Tab查看结果

**4个结果Tab：**

| Tab | 内容 | AI能力 |
|-----|------|--------|
| 💾 备份文件 | 备份文件列表 + 内容预览 | `[🔍AI合规巡检]` |
| 📊 巡检报告 | 巡检报告列表 + 内容预览 | `[🩺AI故障诊断]` |
| 🩺 诊断报告 | AI诊断报告列表 + 预览 | `[🩺AI故障诊断]` |
| 🔍 合规审计 | 审计报告列表 + 预览 | `[🩺AI精审]` |

**文件管理：** 每个Tab左侧列表上方有 `[🔄刷新] [📂打开] [🗑删除]` 按钮。

---

### 🔍 单点运维（单设备全链条）

**用途：** 针对某台具体设备执行全链条运维操作。

**操作流程：**

4. 在下方5个Tab中查看结果和AI分析

**5个结果Tab：**

| Tab | 内容 | AI按钮 |
|-----|------|--------|
| 巡检日志 | 实时巡检输出 | 无 |
| 配置备份 | 备份文件列表 + 内容预览 | `[🔍AI合规巡检]` |

| 🩺 诊断报告 | 诊断报告列表 + 内容预览 | `[🩺AI故障诊断]` |

**AI分析使用：** 切换到配置备份Tab → 选中一个备份文件 → 点击右侧 `[🔍AI合规巡检]` → 自动加载Agent规则分析 → 中文报告。

### 🤖 专家工作站（深度AI分析）

**用途：** 进行深度、多文件、自定义的AI分析，是AI能力最高级入口。

**三种使用方式：**

| 方式 | 操作 | 特点 |
|------|------|------|
| 💬 自由对话 | 选Tab→输入任意问题→点 `▶发送分析` | 完全自定义 |
| 📋 合规审计 | 左侧选文件→点 `[🔍一键合规巡检]`→编辑Prompt→发送 | Agent预填 |

**与嵌入式AI的区别：**
- 支持多文件对比分析
- Prompt可自由编辑
- 支持多轮对话迭代
- 可保存/导出报告

## 四、AI能力速查

### 三层架构

```
第一层：嵌入式按钮 → 单点运维/项目运维 → 一键自动执行
第二层：Agent快捷指令 → 专家工作站 → 可编辑Prompt
第三层：自由对话 → 专家工作站 → 完全自定义
```

### 双层AI分析（V0.2.1 新增）

每次 AI 分析均经历两层处理，确保精准且节省 Token：

```
Layer 1：本地规则引擎（确定性规则，毫秒级）
  → 从巡检报告/配置中提取结构化异常
  → 生成精简异常摘要 + 精准上下文片段
         ↓
Layer 2：AI 大模型（语义分析，深度推理）
  → 仅接收精简摘要 + 异常周边上下文（而非全文）
  → 深度根因分析 + 补充遗漏 + OSI分层排查命令
```

**Token 优化效果**：
- 合规审计：从 18.4KB → 5.0KB（压缩 73%）
- 故障诊断：从 151.6KB → 5.5KB（压缩 96%）

## 六、数据安全

- AI配置加密存储，采用AES-GCM算法
- 密钥绑定机器ID，拷贝到其他电脑需重新配置模型
- 所有操作记录到 `logs/` 目录
- 删除/导出等危险操作有确认对话框
- 批量运维仅执行只读指令（show/display/ping）
- 高危操作（接口启停/路由变更/配置保存）仅生成人工清单

## 七、常见问题

**Q：启动后报错或闪退？**
A：查看 `logs/` 目录下最新日志文件，根据错误信息排查。

**Q：AI按钮点不了，提示"未配置模型"？**
A：进入「模型设置」，确认已保存至少一套模型配置并设为激活状态。

**Q：AI分析结果不理想？**
A：进入「专家工作站」，选择对应Agent快捷指令后可以手动编辑Prompt优化问题描述。

**Q：拷贝到其他电脑AI配置丢失？**
A：正常现象，AI密钥绑定机器ID，需在新电脑上重新配置模型。

**Q：设备连接失败？**
A：先用「单点运维」的 `🧪 测试连接` 功能验证Ping和SSH/Telnet是否能通。

**Q：项目找不到？**
A：检查 EXE 同级目录下 `projects_config.json` 和 `projects/` 目录是否存在。
