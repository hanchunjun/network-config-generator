# NetOps 企业网络自动化运维平台

## 项目概述

**NetOps 企业网络自动化运维平台** 是一款政企级全流程闭环网络自动化运维桌面工具。基于PyQt5框架开发，以四厂商配置脚本生成为基础，叠加多项目标准化管理、全网批量自动化运维、单点设备专项排查、AI智能故障诊断、配置合规审计五大核心能力。

**当前版本：** V1.0 优化版（2026年5月17日）

## 系统要求

- **操作系统**：Windows 10/11（推荐）
- **运行环境**：无需安装Python环境，单EXE直接运行
- **网络要求**：支持SSH/Telnet协议的网络设备
- **硬件配置**：4GB内存以上，500MB硬盘空间

## 快速开始

### 1. 启动程序
```bash
双击 "NetworkConfigGenerator.exe" 直接启动
```

### 2. 创建项目
- 点击"新建项目"，输入项目名称
- 自动生成标准化目录结构
- 在config/device_list.txt中录入设备信息

### 3. 设备配置
- 选择厂商和设备类型
- 填写配置参数，生成配置脚本
- 支持脚本预览、复制和导出

### 4. 批量运维
- 进入"运维工具箱"
- 执行批量配置备份
- 进行全网自动化巡检
- 查看故障设备报告

### 5. AI分析
- 配置AI模型参数（首次使用）
- 选择故障日志进行智能诊断
- 生成配置合规审计报告

## 项目目录结构

### 程序根目录
```
NetworkConfigGenerator/
├── scripts/                   # 公共运维脚本库
│   ├── backup_all_config.py    # 批量配置备份
│   ├── network_inspect.py      # 全网健康巡检
│   └── run_trouble_cmd.py      # 故障二次核查
├── agents/                     # AI代理规则
├── src/                        # 源代码
├── docs/                       # 文档
├── projects/                   # 项目数据目录
├── ai_config.json              # AI配置文件
├── projects_config.json        # 项目配置文件
└── NetworkConfigGenerator.exe  # 主程序
```

### 项目标准结构
```
项目名称/
├── config/
│   └── device_list.txt         # 设备清单
├── output/
│   ├── single_exception/       # 单点设备异常
│   ├── trouble_check_result/   # 故障检查结果
│   └── manual_high_risk_task.txt  # 人工高危操作清单
├── report/                     # AI分析报告
└── config_backup/              # 配置备份
```

## 设备清单格式

`config/device_list.txt` 文件格式：
```
IP地址,厂商,设备类型,用户名,密码,协议,特权密码
192.168.1.1,华为,核心交换机,admin,Huawei@123,ssh,enable123
10.0.0.1,锐捷,接入交换机,ruijie,Ruijie@123,ssh,
172.16.0.1,思科,路由器,cisco,Cisco@123,telnet,
```

**字段说明：**
- **厂商**：华为、锐捷、H3C、思科
- **设备类型**：接入交换机、核心交换机、路由器、AC
- **协议**：ssh 或 telnet
- **特权密码**：可选，部分设备需要

## 安全特性

### 数据安全
- 🔒 设备凭证本地存储，不上传云端
- 🔒 多项目物理隔离，杜绝数据混淆
- 🔒 AI密钥加密存储，安全可控
- 🔒 敏感操作需要人工确认

### 操作安全
- ⚠️ 自动化仅执行只读指令（show/display/ping/traceroute）
- ⚠️ 禁止自动修改配置、启停接口、重启设备
- ⚠️ 高危操作生成人工清单，需要二次确认
- ⚠️ 所有操作记录完整日志，可追溯

## 运维流程

### 标准运维流程
1. **项目创建** → 建立项目目录，录入设备信息
2. **配置生成** → 生成设备开局配置脚本
3. **批量备份** → 全网上线设备配置归档
4. **定期巡检** → CPU/内存/接口状态监控
5. **故障处理** → 异常设备专项排查
6. **AI分析** → 故障诊断与合规审计
7. **报告归档** → 所有报告按项目分类存储

### 应急处理流程
1. 单点设备问题 → 使用"单点设备巡检"
2. 临时接入设备 → 独立录入，不干扰批量流程
3. 故障快速定位 → AI智能分析，七层诊断
4. 处理结果归档 → 自动归入当前项目

## 故障排查

### 常见问题

**1. 设备连接失败**
- 检查网络连通性
- 验证设备IP、端口、协议
- 确认用户名密码正确
- 检查设备SSH/Telnet服务状态

**2. AI功能不可用**
- 检查AI配置是否完整
- 测试API连通性
- 验证API Key有效性
- 查看网络代理设置

**3. 项目切换异常**
- 确认项目路径正确
- 检查设备清单格式
- 验证配置文件权限

### 日志文件
- 程序日志：程序根目录 `*.log` 文件
- 操作日志：项目 `output/` 目录下
- 错误日志：Windows事件查看器
