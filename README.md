# NetOps 企业网络自动化运维平台

## 项目概述

**NetOps 企业网络自动化运维平台** 是一款政企级全流程闭环网络自动化运维桌面工具。基于 PyQt5 框架开发，以四厂商配置脚本生成为基础，叠加多项目标准化管理、全网批量自动化运维、单点设备专项排查、AI智能故障诊断、配置合规审计五大核心能力。

**当前版本：** V0.3.1 工具增强版（2026年5月22日）

**下载地址：** [GitHub Releases](https://github.com/hanchunjun/network-config-generator/releases)

- ✅ 兼容锐捷、华为、H3C、思科全系列网络设备
- ✅ 单EXE免环境安装（约48.5MB），双击直跑，整体拷贝即用

## 快速开始

### 1. 启动程序

双击 `NetworkConfigGenerator.exe` 直接启动，首次运行自动创建目录结构，约3-5秒进入主界面。

> **试用模式**：未激活用户进入主界面后，窗口标题显示 `[试用模式]`，可使用「锐捷接入交换机配置」「📜 命令生成」两个基础工具，其他高级功能需激活后使用。

### 2. 激活软件

1. 点击右上角导航栏「🔓 未激活」按钮（或「关于」→「软件激活」）
2. 复制本机机器码（32位大写字符串）
3. 收到激活码后填入（16位或18位），点击「立即激活」
4. 激活成功后全部功能自动开放

### 3. 创建项目

1. 点击 **📁 新建项目** → `＋ 新建项目` → 输入项目名称
2. 点击 `编辑设备清单` → 按格式录入设备信息
3. 设备格式：`IP，厂商，设备类型，用户名，密码，协议，特权密码`

### 4. 配置AI模型

1. 点击 **⚙ 模型设置**
2. 选择厂商 → 填写Base URL → 填写API Key → 填写模型名
3. 点击 `测试连接` → `保存配置`

### 5. 开始使用

- **🖥 设备配置** → 生成四厂商配置脚本
- **🔍 单点运维** → 单设备全链条排查
- **🔧 项目运维** → 批量备份/巡检
- **🤖 专家工作站** → 深度AI分析

## AI能力三层架构

```
第一层：嵌入式按钮 → 单点运维/项目运维 → 一键自动执行
第二层：Agent快捷指令 → 专家工作站 → 可编辑Prompt
第三层：自由对话 → 专家工作站 → 完全自定义
```

### 双层AI分析

```
Layer 1：本地规则引擎（确定性规则，毫秒级）
  → 从巡检报告/配置中提取结构化异常
  → 生成精简异常摘要 + 精准上下文片段
         ↓
Layer 2：AI 大模型（语义分析，深度推理）
  → 仅接收精简摘要 + 异常周边上下文（而非全文）
  → 深度根因分析 + 补充遗漏 + OSI分层排查命令
```

**Token 优化效果**：合规审计压缩 73%，故障诊断压缩 96%

## 软件激活体系

### 激活码格式

- **16位**：旧格式，永久授权（如 `F1637109D2A857DE`）
- **18位**：新格式，含有效期编码（如 `F1637109D2A857DE05`，末尾 `05` = 1年）
- 两种格式均有效，16位码始终视为永久授权

### 激活方案

| 方案  | 说明                           | 适用场景                 |
| ----- | ------------------------------ | ------------------------ |
| 方案A | 纯离线永久激活，无网络依赖     | 熟人内部、小范围分发     |
| 方案B | 离线使用 + 180天静默黑名单校验 | 对外分发、商用、需可控   |
| 方案C | 管理员独立制码工具             | 管理员统一出码、台账管理 |

### 有效期选项（10档）

| 选项 | 天数 | 选项 | 天数 |
| ---- | ---- | ---- | ---- |
| 永久 | 0    | 半年 | 180  |
| 10年 | 3650 | 季度 | 90   |
| 5年  | 1825 | 月度 | 30   |
| 3年  | 1095 | 周度 | 7    |
| 2年  | 730  | 1年  | 365  |

### 硬件绑定

- 机器码由CPU序列号 + 硬盘物理序列号生成
- 重装系统/更新驱动不改变机器码
- 更换CPU或硬盘需重新申请激活

```
│   ├── projects_config.json            ← 项目列表索引
│   ├── ai_recent_files.json            ← AI最近文件记录
│   └── cmd_templates.json              ← 批量命令模板（明文JSON）
├── activation/                         ← 用户端激活体系
│   ├── license.dat                     ← 激活授权文件（加密）
│   └── bl_check.dat                    ← 黑名单校验时间记录
├── admin_data/                         ← 管理员数据（独立隔离）
│   ├── records.dat                     ← 授权台账（加密）
│   ├── blacklist.txt                   ← 本地黑名单
│   └── backup/                         ← 台账备份目录
├── logs/                               ← 运行日志
│   ├── netops_YYYYMMDD.log             ← 按日期分割
│   └── crash.log                       ← 崩溃日志
├── single/                             ← 单点运维数据（隔离）
│   ├── single_devices.json.enc         ← 单点设备清单（加密）
│   ├── config_backup/                  ← 单点配置备份
│   ├── output/single_exception/        ← 单点巡检异常
│   └── report/                         ← 单点报告
└── projects/                           ← 项目数据（多项目隔离）
    └── 01_项目名称/
        ├── config/device_list.txt      ← 设备清单
        ├── config_backup/              ← 配置备份
        ├── output/                     ← 批量运维输出
        └── report/                     ← 报告归档
```

## 设备清单格式

`config/device_list.txt` 文件格式：

```
IP地址,厂商,设备类型,用户名,密码,协议,特权密码
192.168.1.1,华为,核心交换机,admin,Huawei@123,ssh,enable123
10.0.0.1,锐捷,接入交换机,ruijie,Ruijie@123,ssh,
172.16.0.1,思科,路由器,cisco,Cisco@123,telnet,
```

**字段说明：**

- **厂商**：华为、锐捷、H3C、思科
- **设备类型**：接入交换机、核心交换机、路由器、AC控制器
- **协议**：ssh 或 telnet
- **特权密码**：可选

## 安全特性

### 数据安全

- 🔒 设备凭证本地存储，不上传云端
- 🔒 多项目物理隔离，杜绝数据混淆
- 🔒 AI密钥加密存储（AES-GCM）
- 🔒 激活授权文件AES-GCM加密存储
- 🔒 管理员台账AES-GCM加密存储

### 操作安全

- ⚠️ 自动化仅执行只读指令（show/display/ping/traceroute）
- ⚠️ 禁止自动修改配置、启停接口、重启设备
- ⚠️ 高危操作生成人工清单，需二次确认
- ⚠️ 所有操作记录完整日志，可追溯
